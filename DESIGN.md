# 明白纸 · 视觉身份（v0.1）

## 具体参照

**anhao（家庭暗号）同一套"放大版极简"语言，但把整体音量调低一档**：
暖米白底、深松绿英雄块、茶青胶囊按钮、加粗中文大标题——这些原样继承，
因为两个产品同属"家庭信任工具"家族，用户认出"这是同一家做的"本身就是信任背书。

调低的部分：anhao 面对的是"正在发生的诈骗"，需要紧迫感和一步直达的 SOS 通道；
明白纸面对的是"生前从容整理"，没有任何需要争分夺秒的场景。因此参照物从
"银行长辈版 App 的警觉感"移向**"银行/保险的家属交接文件 + 一份体面的家庭文书"**——
界面依然现代、克制，但语气更平、留白更从容，不设常驻警示条，不设倒计时式引导。

适老化三条硬规矩全部保留：主要文字 ≥18pt、主点击区 ≥60pt、对比度 ≥4.5:1、行距 ≥1.3。

## 各章节理由

- **深松绿英雄块（首页）**：延续 anhao 的第一屏一句话价值主张手法，但文案基调从
  "对抗诈骗"换成"给家人留一张明白纸"——同样的视觉容器，装的是完全不同的情绪。
- **两张入口卡（盘点 / 指南）**：只有两个主入口，不做第三张"平安确认"卡片——
  v0.1 该模块不存在，硬摆一张灰卡片占位是伪需求，YAGNI。
- **三不承诺用"card sand"而非红色警示**：不存密码、不碰资金、不是遗嘱，这三句是
  全产品最重要的免责声明，但它描述的是"工具的边界"而不是"危险"，用 anhao 里
  "要紧的一条"同款暖米色卡片承载，庄重但不吓人。
- **分类卡片用统一茶青图标，不做八色彩虹**：八个类别（社交/支付/理财/游戏/云存储/
  订阅/域名及网站/自媒体账号）若每类配一个专属色相，视觉噪音会盖过内容本身；
  统一用一个标签图标+文字区分，符合"一屏一个主张"的家族审美。
- **凭证线索输入框旁的琥珀色提示文字**：这是本产品红线级别的安全提示（不要写密码），
  用 amber 而非 alert-red——因为这不是"出事了"，是"操作前的常态提醒"，红色留给
  真正不可逆的动作（删除）。
- **打印样式独立于屏幕配色，全部黑白灰**：见下方专门章节。
- **指南卡的可信度徽标用中性灰/浅色而非红黄绿灯**：`unverified` 用灰底文字直陈
  "此信息未经官方渠道核实"，不用警示色——它提示的是信息状态，不是风险等级，
  用红色反而会让用户误读成"这个平台很危险"。

## 打印样式是一等公民

纸质清单是本产品对抗"数字失联"最后也最可靠的介质——设备会锁屏、账号会过期，
一张放在保险柜里的纸不会。因此打印排版不是"把屏幕内容倒进 A4"，而是单独设计：

- **黑白优先**：打印区域（`#printArea`）不使用任何彩色背景块，正文纯黑（`--print-ink`），
  分类标题用细灰线（`--print-rule`）而非色块分隔，条目间用浅灰虚线（`--print-dashline`）。
  理由：大多数家庭打印机是黑白激光/普通喷墨，色块在黑白打印下变成一团脏灰，
  且彩色墨水成本高、家用打印机常年缺色导致偏色——纯黑白排版在任何打印机上都体面。
  这也顺带解决了"复印/传真给亲戚"场景下的可读性。
- **封面式抬头**：标题《家庭数字资产清单》/《XX 家属操作卡》+ 生成日期，
  仿正式文书的抬头做法（粗黑分隔线），拿到手第一眼要像"一份文件"而不是"一张截图"。
  版心边距 `18mm 16mm`，贴近 A4 打印默认安全边距。
  操作卡打印遵循同一套封面规则，标题替换为平台名 + 家属操作卡。
- **末尾免责 + 签名区**：清单打印件末尾固定附"本清单不是遗嘱……"一段文字，
  和两栏签名留白（立表人签名/日期、家属确认签收/日期）——法律效力为零，
  但这个动作本身在心理上完成了"郑重交接"，参照家庭遗物清单/交接单的惯例格式。
- **屏幕上的交互元素在打印时整体隐藏**：按钮、导航、表单一律不出现在纸面上，
  只有 JS 在打印前现拼的 `#printArea` 内容可见（`visibility` 技巧实现，见 index.html）。

## Do's

- 大标题延续 1.8rem+ 800 字重；一屏一个主张，人生大事不需要视觉噪音陪衬。
- 一屏一个主按钮；次要动作用浅底胶囊或幽灵按钮。
- 卡片一律圆角 + 单层浅投影，与 anhao 保持像素级一致的"手感"。
- 任何本地存储行为，当场用一句话说明"只存在这台设备的浏览器里"。
- 涉及密码/凭证的输入框旁，永远配一句琥珀色提醒："不要写密码"。
- 未核实的指南信息，永远原样标注"此信息未经官方渠道核实"，不许美化措辞。

## Don'ts

- UI 文案不用 emoji（家规红线，继承 anhao）；图标只用内联线性 SVG。
- 不用恐吓腔、不渲染死亡场景；"身后""家属""处理意愿"这类中性词，不用"去世""遗体"等直白词。
- 不引入外部字体/图标库/框架/CDN；单文件自包含，双击本地可用。
- 不设常驻警示条（SOS 式底部条）——本产品没有需要"一步直达"的紧急通道，硬加一个是抄 anhao 的形，丢了 anhao 的神。
- 警示红不用于任何"死亡/身后事"相关的文案色块，只允许用于"删除记录/清空数据"这类不可逆操作的按钮文字或确认提示。
- 打印区域不用茶青、深松绿等品牌色块——省墨、防偏色、像正式文件优先于品牌感。

## Taste（取舍与为什么）

- **取向一句话**：anhao 的"稳"继续用，anhao 的"劲"（红色 SOS、来电屏的紧迫感）
  在这里全部收起——本产品的高潮不是"这一秒要不要转账"，而是"这张纸家人看得懂"。
- **关键取舍**：
  - **红色的降级**：BRIEF 明确本产品几乎没有紧急场景，把 alert 从"随处可见的
    警示语言"收窄到"仅删除类不可逆操作的确认文案"，是本 DESIGN 与 anhao 最大的分歧点，
    也是唯一必须写清楚、不能靠"继承家族 DNA"蒙混过去的决定。
  - **不做长辈大字开关**：anhao 的 elder 模式服务"长辈自己用手机操作核验/演习"的场景；
    明白纸的模块 1（盘点）大概率由子女/本人代填，模块 2（指南）虽可能被长辈在悲伤中翻阅，
    但基础字号已是 19px、行距 1.6，且是"查资料"而非"高压操作"，暂不值得为一个开关
    再引入一套 rem 缩放和状态持久化——YAGNI，v0.2 若用户反馈字太小再加。
  - **可信度标注用中性灰而非红黄绿灯**：红黄绿灯是"风险仪表"的视觉语言，会让用户
    误以为平台本身有风险等级；实际标注的是"这条信息我们有没有查实"，中性灰底更准确，
    也避免和"删除类"的克制红色抢戏。
  - **打印黑白化**：否决了"打印也保持品牌色"的直觉选项——品牌感应该来自版式和文字，
    不是色块；家用打印机的现实（多为黑白、常年缺色）比品牌一致性更优先。
- **反 slop 清单**：紫蓝渐变、彩虹色分类图标、进度环仪表盘、红黄绿风险灯、
  "见证爱与传承"式煽情文案、毛玻璃面板、emoji 标题、打印页面上的品牌色块。
